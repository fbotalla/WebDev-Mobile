Functions: There is a big differentece beween referencing a function and calling a function.
The way we call a function is by using the () parenthesiss at the end of the function name or variable name
If this is not done, the result is going to be a simple reference to the function. This is basically the concept
of a pointer. Here is an exmaple: 

        const testFunction = () =>{
             console.log("Hello world!");
        }
        

        const pnter = testFunction; //The result here is as  simple reference to the anonymous function
        pnter() // this is going to console log "Hello world!"
        


        const testFunction = (...values) =>{} // this accepts any number of arguments and puts them in an array of values.



Callbacks are one of the strenght of javaScript. They are heavily relied upon because of the ability
that the developer has to run a certain code in repsonse to a certain event with a certain parameter.
Currently I am working on my Senior Project and I rely heavily on callback to architecture my logic.


There are many ways to loop through an array in JavaScript. One that interested me especially was the .map(). This is
systematically different from other ways of looping because .map() takes a callabck function as a parameter. This means
that for each element in result of the loop, the callback is called on that value.

The spread operator ... becomes very useful when I want to work with an array, but I dont not want to rewrite
it's content each time something is added or deleted. This spread operator ... makes the array very dynamical and useful
especilyl when queryin from APIs or Databases.

The date structure SET is very useful when we don not want to have any duplicate whatsoever. In the background of a set,
the set checks for the value attempted to be inserted against all the values currently in the set. If this return true (meaning there is a match)
then the adding operation is complely ignored.

The nice thing about Logic in programming languages is that it does not change. The syntax might, but the logic will not.
Therefore the concept of looping and coditional statements remains the same across all programming lanauges, you basically only need to learn it once.
